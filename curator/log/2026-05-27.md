### Learnings (candidates for playbook consolidation)

- The big value of competitor-harvest is **venue + facilitator discovery**, not event volume. First run: 2 events, 3 venues, 2 facilitators.
- Blissbase exposes outbound ticket URLs **inside the description text**, not as structured `<a href>` links. The harvester needs substring scans for `megatix.co.id`, `eventbrite.com`, `tickettailor.com`, `lu.ma`, `allevents.in`.

- ToDo.Today's Cloudflare wall is a real blocker. Try `curl -A '<realistic user-agent>'` or a public WhatsApp channel fallback. Don't burn time fighting headless Playwright.
- Attribution audit: 0 aggregator URLs in `source_url` or `external_ticket_url` fields in today's inbox. Spot-check this every run — if a row leaks `.blissbase.app` / `.soulwise.io` / `.todo.today` in either URL column, that's a P0 bug.
